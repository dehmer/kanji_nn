To detect the exact boundary where deliberate Kanji writing ends and the "dirty" pre-pen-up phase begins using a Change Point Detection (CPD) mean-diff sweep, the most promising combined weighted sum signal $S_t$ focuses on the inverse relationship between pen pressure and kinematic instability.
The optimal combined signal formula is:
$$S_t = w_1 \cdot (1 - P_t) + w_2 \cdot v_t + w_3 \cdot \Delta s_t$$
Where:

* $P_t$ is the normalized pressure ($0 \le P_t \le 1$).
* $v_t$ is the central speed.
* $\Delta s_t$ is the incremental displacement magnitude (ds).
* $w_1, w_2, w_3$ are positive weights that must be normalized based on the variance of each feature.

------------------------------

## 1. Feature Engineering and Transformation
Before feeding the signals into a mean-diff sweep, they must be transformed so that a transition into the pre-pen-up phase causes a sharp, simultaneous positive shift in the mean.

* Inverted Pressure ($1 - P_t$): Deliberate writing maintaining steady, higher pressure. During the pre-pen-up phase, the user lifts the stylus, causing pressure to drop rapidly toward zero. Inverting this ensures that the signal spikes upward as pressure drops.
* Central Speed ($v_t$): As the hand prepares to lift the stylus, intentional stroke mechanics collapse. This often results in a rapid, uncontrolled acceleration or a sudden erratic jerk.
* Displacement Magnitude ($\Delta s_t$): Micro-tremors and flicking motions right before the pen leaves the digitizer cause erratic sample spacing. Because $\Delta s_t$ is calculated via np.linalg.norm(np.diff(...)), any sudden whip or stabilization loss creates a massive jump in this metric.

## 2. Standardization and Weighting Strategy
A mean-diff sweep is highly sensitive to the variance of its input signal. If one feature has a naturally larger scale, it will dominate the CPD. You must normalize the features before combining them.

```python
import numpy as np
# 1. Transform features so they all increase during pre-pen-up
f1 = 1.0 - pressure           # Inverted pressure
f2 = central_speed            # Speed spikes
f3 = np.append(ds, ds[-1])    # Match dimensions with stroke array

# 2. Standardize via Z-score to equalize variance

f2_norm = (f2 - np.mean(f2)) / (np.std(f2) + 1e-6)
f3_norm = (f3 - np.mean(f3)) / (np.std(f3) + 1e-6)

# Pressure drop is the most reliable anchor for lift-off
w1, w2, w3 = 0.5, 0.25, 0.25
S = (w1 * f1_norm) + (w2 * f2_norm) + (w3 * f3_norm)
```

## 3. Execution of the Mean-Diff Sweep
The mean-diff sweep acts as a moving two-window t-test. It maximizes the absolute difference in means between a window before index $t$ and a window after index $t$.
$$\text{Score}(t) = \left\vert{} \frac{1}{k}\sum_{i=t-k}^{t-1} S_i - \frac{1}{k}\sum_{i=t}^{t+k-1} S_i \right\vert{}$$
Because the pre-pen-up phase occurs at the very end of the stroke, you should set your window size $k$ to be relatively small (e.g., 5 to 10 samples) and focus your sweep exclusively on the final $25\%$ of the stroke data to avoid false positives from earlier deliberate stroke pauses.

import numpy as np

# 1. Feature Combination & Z-Score Standardization Engine
def compute_combined_signal(stroke_data, w1, w2, w3):
    """
    Transforms and standardizes features so they align to a positive mean shift.
    stroke_data: dict containing 'pressure', 'central_speed', and 'ds' arrays.
    """
    # Align shapes: pads 'ds' by repeating last element to match pressure/speed
    ds_padded = np.append(stroke_data['ds'], stroke_data['ds'][-1])

    # Transform so all features increase during the dirty pre-pen-up zone
    f1 = 1.0 - stroke_data['pressure']
    f2 = stroke_data['central_speed']
    f3 = ds_padded

    # Equalize scales via Z-score normalization
    f1_n = (f1 - np.mean(f1)) / (np.std(f1) + 1e-6)
    f2_n = (f2 - np.mean(f2)) / (np.std(f2) + 1e-6)
    f3_n = (f3 - np.mean(f3)) / (np.std(f3) + 1e-6)

    # Return weighted sum signal
    return (w1 * f1_n) + (w2 * f2_n) + (w3 * f3_n)

# 2. Mean-Difference Sweep CPD Engine
def find_change_point(S, window_size=6):
    """
    Finds the index that maximizes the change in mean between two adjacent windows.
    Focuses exclusively on the last 40% of the stroke to avoid writing pauses.
    """
    n = len(S)
    scores = np.zeros(n)
    start_search_idx = int(n * 0.6)

    for t in range(start_search_idx, n - window_size):
        left_window = S[t - window_size : t]
        right_window = S[t : t + window_size]

        # Maximize absolute difference in local means
        scores[t] = abs(np.mean(left_window) - np.mean(right_window))

    return np.argmax(scores)

# 3. Supervised Weight Optimization Loop
def optimize_weights(labeled_dataset):
    """
    Performs a 3D grid search to find weights that minimize sample-distance MAE.
    labeled_dataset: list of dicts with stroke arrays and an 'exact_boundary_idx'
    """
    best_mae = float('inf')
    best_weights = None

    # 0.1 step resolution for weights
    grid = np.linspace(0, 1, 11)

    for w1 in grid:
        for w2 in grid:
            w3 = 1.0 - w1 - w2
            if w3 < -1e-6:
                continue
            w3 = max(0.0, w3) # Sanitize floating-point errors

            total_error = 0
            for stroke in labeled_dataset:
                # Generate custom combined signal for this weight triplet
                S = compute_combined_signal(stroke, w1, w2, w3)
                # Run the sweep
                predicted_idx = find_change_point(S, window_size=6)
                # Track error
                total_error += abs(predicted_idx - stroke['exact_boundary_idx'])

            mae = total_error / len(labeled_dataset)

            if mae < best_mae:
                best_mae = mae
                best_weights = (round(w1, 2), round(w2, 2), round(w3, 2))

    return best_weights, best_mae

import numpy as np

# 1. Feature Combination & Z-Score Standardization Engine
def compute_combined_signal(stroke_data, w1, w2, w3):
    """
    Transforms and standardizes features so they align to a positive mean shift.
    """
    ds_padded = np.append(stroke_data['ds'], stroke_data['ds'][-1])

    f1 = 1.0 - stroke_data['pressure']
    f2 = stroke_data['central_speed']
    f3 = ds_padded

    # Z-score normalization handles varying global scales across different strokes
    f1_n = (f1 - np.mean(f1)) / (np.std(f1) + 1e-6)
    f2_n = (f2 - np.mean(f2)) / (np.std(f2) + 1e-6)
    f3_n = (f3 - np.mean(f3)) / (np.std(f3) + 1e-6)

    return (w1 * f1_n) + (w2 * f2_n) + (w3 * f3_n)

# 2. Adaptive Mean-Difference Sweep CPD Engine
def find_change_point_adaptive(S, window_pct=0.08, min_w=4, max_w=12):
    """
    Finds the change point using a window size that scales with stroke length.

    window_pct: Percentage of total stroke length to use for window size.
    min_w: Hard minimum window size (prevents noise sensitivity on short strokes).
    max_w: Hard maximum window size (prevents signal bleeding on long strokes).
    """
    n = len(S)
    scores = np.zeros(n)

    # Calculate adaptive window size based on individual stroke length
    window_size = int(round(n * window_pct))
    window_size = max(min_w, min(window_size, max_w))

    # Focus exclusively on the last 35% of the stroke to find the pre-pen-up zone
    start_search_idx = int(n * 0.65)

    for t in range(start_search_idx, n - window_size):
        left_window = S[t - window_size : t]
        right_window = S[t : t + window_size]

        # Maximize the difference between the two adaptive windows
        scores[t] = abs(np.mean(left_window) - np.mean(right_window))

    return np.argmax(scores)

# 3. 4D Joint Optimization Loop (Weights + Adaptive Window Scale)
def optimize_pipeline(labeled_dataset):
    """
    Optimizes both the feature weights and the adaptive window percentage.
    """
    best_mae = float('inf')
    best_params = {}

    weight_grid = np.linspace(0, 1, 11)
    # Search from 4% to 14% of stroke length for the optimal window scale
    pct_grid = np.linspace(0.04, 0.14, 6)

    for w1 in weight_grid:
        for w2 in weight_grid:
            w3 = 1.0 - w1 - w2
            if w3 < -1e-6:
                continue
            w3 = max(0.0, w3)

            for pct in pct_grid:
                total_error = 0
                for stroke in labeled_dataset:
                    S = compute_combined_signal(stroke, w1, w2, w3)

                    # Run CPD with current adaptive window percentage
                    predicted_idx = find_change_point_adaptive(S, window_pct=pct)
                    total_error += abs(predicted_idx - stroke['exact_boundary_idx'])

                mae = total_error / len(labeled_dataset)

                if mae < best_mae:
                    best_mae = mae
                    best_params = {
                        'weights': (round(w1, 2), round(w2, 2), round(w3, 2)),
                        'window_pct': round(pct, 3)
                    }

    return best_params, best_mae

## Why This Resolves Variable Stroke Lengths

* The Scale Invariance Problem: A fixed window of 10 samples works great on a 150-sample long sweep, but it will completely break and overflow the valid boundaries on a tiny 40-sample stroke (like a short Kanji dot/ten).
* Min/Max Boundary Protection: Line 32 clamps the window size using min_w and max_w. This ensures that even if a stroke is massive (e.g., 400 samples), the window won't scale up to an absurdly wide 32 samples, which would ruin accuracy by smoothing over the very sudden, sharp lift-off transition.
